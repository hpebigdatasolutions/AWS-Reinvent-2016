AWS Reinvent 2016 Notes

View project on GitHub

Overview

Overall, the conference was huge - number of attendees, sessions, new feature announcements, impressions, and so on. Obviously, there was a lot of koolaid, designed to pull one into the AWS fold. But at the same time it was impressive to see the decidedly educational and cost saving angle of many of the sessions.

This is an attempt to consolidate notes and impressions (including useful external links) from the conference.

Day 1

Well, officially the Day 1 was on Monday, but Vijay, Rob and I arrived in Las Vegas on Mon evening, so for us this was really Tuesday.

After the registration and breakfast (an impressive coordination of caterers serving 10,000s of attendees), my first session was "Leading Enterprise Innovation at Startup Speed", presented by Eric Tachibana (AWS Prof Services APAC area manager).

Recording of the session -

What I liked about this session was that it related to various quality and process improvement movements of the past (Toyota, Six Sigma), and discussed lean startups and lean enterprises of the future. I recommend that people managers and leaders listen to this.

I have more detailed notes of this, but the key points of accepting that you don't know what the customer wants if you are truly innovating, testing and measuring smart, and iterating at speed sounded very compelling.

Followups - check out the Lean Startup book and the Lean Analytics book.

Meanwhile, Vijay and Rob were at the AWS Partner Keynote, which was reportedly also very good

Check out 41:30, then 48:35, then 56:26 - very neat insights from James Hamilton, AWS Distinguished Engineer.

My next session was Tips for Building Successful Solutions with AWS Marketplace and AWS Quick Start -

For me, this was somewhat underwhelming. I was interested in this topic as I wanted to know more about how our solutions could be sold through AWS Marketplace (see here for some recent developments in this space, including Druva publishing their solution). Unfortunately the session was somewhat too basic and plagued with autoadvancing slides. Still, some useful information.

After lunch, it was on to Getting the Most Bang for Your Buck with EC2

Though we are already using the reserved instances in our AWS environments, this session had good information on how to combine these with the spot instances, and various usage patterns depending on use cases.

Takeaways / followups -

  • Use all 3 types - on demand, reserved, spot
  • Measure, monitor, improve
  • Invest in good tagging schemes to understand usage, pricing, etc
  • Investigate using spot instances for Jenkins (Spot3)
  • Look at Lyft's Jenkins Docker build system (using spot instances)
  • Even DBs could be built dynamically for eg daily reporting and use spot instances
  • (Important) Shut down environments when they are not used to save!
  • Investigate convertible reserved instances

    • My final session of the day was Chalk Talk: Ingesting, Storing, Processing and Extracting Value from Real-Time Big Data.

    There is no recording of this session, but basically it was a dive into using various AWS components (Kinesis, Machine Learning, etc) to solve the use case of tracking airplanes, identifying suspicious flights, and detecting possible collisions between airplanes (using radar data).

    Next session - Deep Dive on Amazon RDS:

    Highlights:
  • Reminder that RDS means OS cannot be accessed / parameters modified
  • Size limits (4 TB SQL Server, 6TB MySQL, Oracle, PG, 64TB Aurora
  • Aurora now PG compatible (in addition to being MySQL compatible)
  • Security and encryption (networking, IAM, all engines support SSL encryption on the fly; also various mechanisms for encryption at res, also KMS)
  • Also various certs (ISO, Fedramp, HIPAA, PCI for DB specifically)
  • Useful tips on KMS with RDS
  • Metrics and Monitoring (CloudWatch, plus also Enhanced Monitoring which provides additional 50 OS level metrics)
  • Also new is Performance Insights for RDS (identifies perf bottlenecks - top SQL, etc) - to be rolled out in 2017
  • High Availability - Standby replicas in multi AZ deployments, and DNS failover at 24:30
  • Read replicas to bring data closer to users and speed up recovery
  • Aurora has some advantages - described at 27:50
  • Scaling RDS - at 30:50
